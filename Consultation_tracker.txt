You are building a feature called **Consultation Tracker PDF Export** for a health application named Aarogyaan.

The goal is to create a system where users can store consultations with multiple sessions, attach documents, and export a clean, doctor-friendly PDF that includes all relevant information and summaries.

---

# 🎯 Overall Objective

Design a system where:

* Users can create consultations and sessions
* Each session contains symptoms, medications, notes, and uploaded documents
* The system generates a **fully structured PDF report**
* The PDF includes:

  * session details
  * embedded document previews (not links)
  * AI-generated summaries for each session
  * proper branding (logo, header, footer)

The most important requirement is:
👉 **The PDF should be generated in advance so that export is instant.**

---

# 🧠 Core System Approach

## 1. Do NOT generate everything at export time

Avoid generating PDFs or processing documents when the user clicks export.

Instead:

* Perform heavy processing earlier (in the background)
* Keep everything ready beforehand

---

## 2. Document Handling Strategy

Whenever a user uploads a document:

* Fetch the actual file (not just the link)
* Convert it into images (all formats should become images)
* Store these processed images for reuse

This ensures:

* All documents can be embedded visually inside the PDF
* No dependency on external links
* Faster PDF generation later

---

## 3. AI Summary Strategy

Whenever a session is created or updated:

* Analyze session data (symptoms, medications, notes)
* Extract useful text from uploaded documents (if needed)
* Generate a structured summary of the session

The summary should include:

* key symptoms
* treatment given
* important findings
* progression (improving, worsening, unchanged)
* a short readable summary

Important rules:

* Do not invent medical information
* Only summarize what is present
* Keep output structured and consistent

Store this summary so it can be reused later.

---

## 4. PDF Generation Strategy

Whenever anything changes in a consultation (new session, edits, document changes):

* Mark the existing PDF as outdated
* Regenerate a completely new PDF in the background
* Replace the previous PDF with the new one

Important:
👉 Do NOT try to edit an existing PDF
👉 Always rebuild it from scratch using stored data

---

## 5. PDF Content Structure

The generated PDF should include:

* App logo at the top (header on every page)
* Consultation title and basic info
* For each session:

  * symptoms
  * medications
  * notes
  * AI summary (clearly highlighted)
  * embedded document images (inline preview)
* Footer:

  * “Generated by Aarogyaan”
  * page numbers
  * timestamp
* Disclaimer:

  * AI-generated summaries should not replace medical judgment

---

## 6. Storage Strategy

* Store only ONE final PDF per consultation (latest version)
* Always overwrite the previous PDF instead of creating new copies
* Use a consistent path so there are no duplicates or naming conflicts

---

## 7. Handling Updates and Edge Cases

When a user updates a session:

* Immediately mark PDF as “processing”
* Start regeneration in the background

If a user tries to export while processing:

* Either return the last available PDF
* Or wait briefly and return the updated one if ready

Ensure:

* no broken downloads
* no missing files
* no inconsistent state

---

## 8. Performance Principles

* All heavy tasks (document conversion, AI summaries, PDF generation) should run in the backend
* Use background processing wherever possible
* Do not block the user interface

The goal is:
👉 PDF export should feel instant (1–2 seconds max)

---

## 9. Key Design Principles

* PDF is a derived output, not the source of truth
* Always regenerate instead of modifying
* Preprocess everything early
* Store reusable results (images, summaries)
* Handle system state carefully (ready vs processing)

---

# ✅ Expected Outcome

After implementation:

* Users can export consultation PDFs instantly
* Doctors can quickly understand sessions via summaries
* All documents are visible directly inside the PDF
* The system is fast, scalable, and reliable

---

Build the system following these principles and ensure clean separation between:

* document processing
* AI summarization
* PDF generation
* background task handling
